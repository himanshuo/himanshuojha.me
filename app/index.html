SpamAdmin

Spam Management System for Comments and Projects

Spam Admin is a spam management system. It is two-part.

First is an Asynchronous Tornado web service that determines whether the input is spam or not. It accepts a message and whatever metadata can be provided along with the message. This can be in either JSON or file format. It returns SPAM or HAM, as appropriate. You can optionally get a full report about why the message was marked as spam or ham. You can also optionally train the web service by providing definite spam or definite ham to allow it to make better decisions in the future. It also automatically trains itself if a message is very significantly spam or very significantly ham. This is built atop the popular Spam Assasssin email spam detection technology, customized to work with all messages.

Second is a administration page for admin's to quickly verify whether comments or projects at the Center for Open Science's Open Science Framework (OSF) are spam. Comments and projects sent here are not only from the web service but from other events in the OSF that can flag a comment or project. Once a admin makes a decision on the validity of a comment or project, the Tornado web service is automatically trained appropriately.

HoosClass - Alerts for UVA Courses

A web application that allows you to sign up and recieve alerts when information on UVA courses open up.The server side is written in Node.js. It allows for scraping Lous List to attain data, running cron-like jobs to update data, and has REST endpoints to be used by the front end. I used websockets to transfer data between client side and server side and sending out emails. The client side is written in AngularJS. It involves creating custom Angular directives, controllers, services, and filters. All data is persisted using Postgre SQL.
